# resolver

Учебный резолвер зависимостей на Rust. Читает манифест проекта `project.toml`
и локальный реестр пакетов `registry.json`, строит граф зависимостей,
проверяет его на циклы и конфликты версий и печатает порядок установки.

## Сборка

```
cargo build --release
```

## Запуск

```
cargo run -- project.toml registry.json
```

## Тесты

```
cargo test
```

## Пример вывода

```
Порядок установки:
  1. serde_derive 1.0.0
  2. serde 1.0.0
  3. tokio 1.2.0
  4. my_app 1.0.0
```

## Что делает программа

- читает манифест в формате TOML;
- подтягивает метаданные пакетов из локального JSON-реестра;
- строит ориентированный граф зависимостей;
- ищет циклы (например, `A -> B -> A`) через DFS с тремя цветами;
- ищет конфликты версий (разные пакеты требуют разные версии одного и того же);
- выдаёт топологически отсортированный список пакетов (алгоритм Кана).

## Структура

- `src/manifest.rs` — структуры и парсинг `project.toml`
- `src/registry.rs` — структуры и парсинг `registry.json`
- `src/graph.rs` — граф зависимостей, поиск циклов и топсорт
- `src/error.rs` — общий тип ошибок `ResolverError`
- `src/resolver.rs` — основная логика
- `src/main.rs` — CLI
